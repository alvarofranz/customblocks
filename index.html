Introduction

The way to create a WordPress block has changed a couple of times since they started existing. That's normal because WordPress is a natural project that grows organically. Everything changes. But don't worry. Here is a step by step, updated guide.

Step 1: Have a local environment with node installed

If you are going to develop a block, chances are you know your way around, and just want the steps to create a block, but already have a local environment set up and know what node and npm is.

If you do, go to the next step.

If you don't, just keep reading.

Local WordPress environment

Don't panic. This isn't anything crazy. The point is... you need the tools to run WordPress on the machine you are using to develop your block. I guess that makes sense. If you can't run WordPress, you can't look at the block you are creating.

All you need is to install Docker and Node.

  • Docker: Install on Windows, Linux or macOS .
  • Node: Install Node.

Once this is done, just open a terminal. A terminal is a way to talk to your computer with commands. You can open a terminal as follows:

  • Windows: Ctrl + Alt + T
  • macOS: Cmd + Space, then type "terminal"
  • Linux: You already know ;)

Once it's open, you will see a window where you can type in a text. That's where you can write or copy/paste the following command (a command is an instruction for your computer):

npm -g i @wordpress/env

This basically means: "Hey, please install the package from npm called @wordpress/env".

Hit intro and let it work. Once that's done, you should see something like this or very similar:

Great news, now you can start your local environment by running:

wp-env start

You should now see something as follows:

There are two instances... just open the first link, and you should see your WordPress installation up and running!

You can now access the admin area by appending /wp-admin to the URL. The default credentials are:

  • Username: admin
  • Password: password

Good to go!, you have your local WordPress environment set up. Now you can start coding plugins, themes, blocks, and even add cool new stuff and contribute to the community!

Step 2: Use the create-block tool

The create-block tool is an official npm package used to create WordPress blocks.

To use it just run:

npx @wordpress/create-block@latest my-custom-block

Of course, you can change "my-custom-block" to whatever you like. This is the name of your new block.

If you never installed the create-block package before, it will ask you to install it. But the cool thing about using npx is that it will clean everything up after using it. Also, note that using @latest, ensures we will use the most up-to-date version.

Just enjoy the process while it runs.

What this command does is:

  • It creates a WordPress plugin for you
  • That plugin already contains the base for the new block

Once completed, it should say: "Done: WordPress plugin My Custom Block bootstrapped in the my-custom-block directory". And also give you a bunch of example commands, like:

  • npm start: Starts the build for development.
  • npm run build: Builds the code for production.

And some others... but yeah for now, what you should do is look at the plugin that has been created for you, which is inside the WordPress plugins folder and has a structure as follows:

You already created a custom block plugin, and can now go to your admin area to activate it.

Once activated, your block is now available in the editor. For example, you may create a new post and there you will see it, ready to be used!

Click the "+" symbol to add a block and either browse the gallery or just type the name of the block, which in this case is "My custom block".

When you add it to the page, you will see the block in action.
